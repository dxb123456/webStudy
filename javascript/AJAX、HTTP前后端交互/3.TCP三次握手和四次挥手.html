TCP:(传输控制协议)面向连接的协议 1.A=>B 是否能发送数据请求建立连接；包内容（同步信息SYN=1 和序列号seq=X） 2.B=>A 可以 什么时候发 ；应答内容ACK=(X+1)和同步序列号SYN=1 和seq=Y; 3.A=>B: 确认应答，现在就发 : ACK=（Y+1）+ seq=(Z) 三次对话的目的是：使数据包的发送和接收同步，经过三次后才能正常发送数据 seq:序列号 根据序列号识别某个操作 ACK：确认标志 --确认编号 SYN：同步标志 FIN：结束标志 A:主动方 B:被动方 1.A =>B 是否可以结束 发送FIN + ACK +seq=X 2.B =>A 先应答 发送ACK=(X+1) + seq=Z 3.B=>A 发送结束请求 发送FIN + ACK=X + seq=Y 4:A=>B 确认结束 发送ACK=（Y+1）+ seq=X UDP：(用户数据报协议)非连接的协议 1. 传输数据之前，源端和终端不建立连接 2. 信息包的标题很短，只有8个字节，相比TCP20个字节开销更小 3.传输速度快，易丢包
